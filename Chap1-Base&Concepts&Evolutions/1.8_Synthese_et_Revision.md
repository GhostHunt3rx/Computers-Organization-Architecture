## Glossaire complet des termes clés

| Terme | Définition synthétique |
|---|---|
| **Architecture (ISA)** | Attributs visibles par le programmeur qui impactent l'exécution logique des programmes |
| **Organisation** | Unités opérationnelles et interconnexions réalisant les spécifications architecturales (invisible pour le programmeur) |
| **CPU** (Central Processing Unit) | Composant qui récupère et exécute les instructions ; comprend ALU, unité de contrôle et registres |
| **ALU** (Arithmetic & Logic Unit) | Effectue les opérations de traitement de données : arithmétiques et logiques |
| **Unité de Contrôle** | Contrôle l'opération du CPU et donc de tout l'ordinateur |
| **Registres** | Emplacements de stockage internes ultra-rapides du CPU |
| **Mémoire Principale** | Stocke données et instructions accessibles par le CPU |
| **Bus Système** | Mécanisme d'interconnexion entre CPU, mémoire principale et I/O |
| **Cœur** (Core) | Unité de traitement individuelle sur une puce processeur multicœur |
| **Processeur** (Processor) | Pièce physique de silicium contenant un ou plusieurs cœurs |
| **Multicore Processor** | Processeur contenant plusieurs cœurs sur une seule puce |
| **Cache Mémoire** | Mémoire intermédiaire plus petite et plus rapide placée entre CPU et mémoire principale |
| **Concept Programme Stocké** | Idée fondamentale de von Neumann : données ET instructions stockées dans la même mémoire |
| **Porte** (Gate) | Dispositif implémentant une simple fonction booléenne/logique |
| **Cellule Mémoire** | Dispositif capable de stocker 1 bit de données dans 2 états stables |
| **Transistor** | Brique élémentaire des circuits numériques : interrupteur électronique en silicium |
| **Circuit Intégré** (IC) | Circuit entier fabriqué sur un minuscule morceau de silicium |
| **Puce** (Chip) | Pièce de silicium semiconducteur sur laquelle sont gravés des circuits |
| **Motherboard** (Carte mère) | Carte de circuits imprimés principale d'un ordinateur |
| **MCM** (Multichip Module) | Boîtier contenant plusieurs puces nues sur un substrat commun |
| **Loi de Moore** | Le nombre de transistors sur une puce double tous les ~18 mois |
| **x86** | Architecture de processeur Intel introduite avec le 8086 (1978), toujours dominante sur PC/serveurs |
| **CISC** | *Complex Instruction Set Computer* : jeu d'instructions complexe (ex: x86) |
| **RISC** | *Reduced Instruction Set Computer* : jeu d'instructions réduit et régulier (ex: ARM) |
| **ARM** | Architecture RISC dominante dans l'embarqué et le mobile (milliards de puces/an) |
| **Cortex-A/R/M** | Les trois profils ARM : Application, Real-time, Microcontroller |
| **Système Embarqué** | Utilisation d'électronique et logiciel dans un produit (non ordinateur général) |
| **Microcontrôleur** | Puce unique contenant CPU, RAM, ROM/Flash et I/O — « ordinateur sur une puce » |
| **IoT** (Internet des Objets) | Interconnexion croissante de milliards d'objets intelligents via Internet |
| **Deeply Embedded** | Système embarqué à microcontrôleur, non programmable post-déploiement, sans interaction utilisateur |
| **MMU** | *Memory Management Unit* : gère mémoire virtuelle et pagination (Cortex-A) |
| **MPU** | *Memory Protection Unit* : protège régions mémoire critiques sans mémoire virtuelle (Cortex-M/R) |
| **Architecture Harvard** | Architecture à bus séparés pour instructions et données (Cortex-M3) |
| **Architecture von Neumann** | Architecture à bus unique pour instructions et données (ordinateurs classiques) |

## Réponses aux questions de révision

### Q1.1 — Distinction entre organisation et architecture informatique

L'**architecture informatique (ISA)** désigne les attributs du système visibles par le programmeur : le jeu d'instructions, les types de données, les mécanismes I/O et les techniques d'adressage mémoire. Elle définit *ce qu'on peut programmer*.

L'**organisation** désigne les unités opérationnelles concrètes et leurs interconnexions qui réalisent ces spécifications architecturales : les signaux de contrôle, les interfaces vers les périphériques, la technologie mémoire. Elle détermine *comment c'est implémenté*.

> **Exemple** : l'existence d'une instruction `MULTIPLY` est une décision **architecturale** ; son implémentation via une unité dédiée ou des additions répétées est une décision **organisationnelle**.

---

### Q1.2 — Distinction entre structure et fonction d'un ordinateur

La **structure** désigne la manière dont les composants sont interconnectés — leurs relations physiques et logiques.

La **fonction** désigne l'opération de chaque composant individuel dans le cadre de cette structure.

Pour concevoir ou comprendre un système, on doit analyser les deux : la structure explique *l'architecture du système*, la fonction explique *ce que fait chaque partie*.

---

### Q1.3 — Les quatre fonctions fondamentales d'un ordinateur

1. **Traitement de données** : opérations arithmétiques et logiques sur les données
2. **Stockage de données** : mémoire à court terme (RAM) et long terme (fichiers)
3. **Mouvement de données** : I/O avec périphériques et communications de données
4. **Contrôle** : orchestration de toutes les parties par l'unité de contrôle en réponse aux instructions

---

### Q1.4 — Composantes structurelles principales d'un ordinateur

- **CPU** : contrôle et traitement de données
- **Mémoire principale** : stockage de données et instructions
- **I/O** : déplacement de données entre ordinateur et environnement extérieur
- **Interconnexion système (bus)** : communication entre les trois précédents

---

### Q1.5 — Composantes structurelles d'un processeur

- **Unité de contrôle** : contrôle l'opération du CPU
- **ALU** : effectue les opérations de traitement de données
- **Registres** : stockage interne au CPU
- **Interconnexion CPU (bus interne)** : communication entre contrôle, ALU et registres

---

### Q1.6 — Qu'est-ce qu'un ordinateur à programme stocké ?

Un ordinateur à programme stocké est un ordinateur dans lequel **les instructions du programme et les données sont stockées dans la même mémoire**. Cette idée, développée par von Neumann (et Turing), permet à l'ordinateur de :
- Modifier ses propres programmes
- Être reprogrammé pour différentes tâches
- Traiter les instructions comme des données

C'est la base de l'**architecture de von Neumann** qui équipe pratiquement tous les ordinateurs modernes.

---

### Q1.7 — La loi de Moore

Formulée par **Gordon Moore** en 1965, la loi de Moore observe que le nombre de transistors pouvant être placés sur une seule puce **double environ tous les 18 mois** (initialement chaque année).

Ses conséquences profondes :
- Réduction du coût de la logique et de la mémoire
- Augmentation de la vitesse de fonctionnement
- Miniaturisation
- Réduction de la consommation d'énergie
- Amélioration de la fiabilité

Elle a guidé les investissements et les attentes de l'industrie informatique pendant **plus de 50 ans**.

---

### Q1.8 — La caractéristique distinctive clé d'un microprocesseur

Le microprocesseur est une **puce unique** qui intègre toutes les fonctions d'un CPU (*Central Processing Unit*) — notamment l'ALU, l'unité de contrôle et les registres.

À mesure que la densité des transistors a augmenté, les microprocesseurs ont pu incorporer :
- Des jeux d'instructions plus complexes
- De la mémoire cache
- Finalement plusieurs processeurs (cœurs) sur la même puce
