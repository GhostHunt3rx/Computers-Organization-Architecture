# 1.3 — L'Ordinateur IAS : Premier Modèle Général

> La première génération d'ordinateurs utilisait des **tubes à vide** comme éléments de logique numérique et de mémoire. L'ordinateur **IAS** est probablement le plus célèbre de cette génération et illustre de nombreux concepts fondamentaux présents dans tous les systèmes informatiques.

---

## Le concept de programme stocké (Stored-Program Concept)

L'approche de conception fondamentale implémentée dans l'IAS s'appelle le **concept de programme stocké**. Cette idée est généralement attribuée au mathématicien **John von Neumann**, bien qu'**Alan Turing** l'ait développée approximativement au même moment.

### Chronologie
- **1945** : Première publication — proposition de von Neumann pour l'EDVAC (*Electronic Discrete Variable Computer*)
- **1946** : Début de la conception de l'IAS, à l'Institut for Advanced Studies de Princeton
- **1952** : Achèvement de l'IAS
- C'est le **prototype de tous les ordinateurs à usage général** qui ont suivi

---

## Les 5 parties de l'ordinateur selon von Neumann (1945)

| Partie | Nom | Rôle |
|---|---|---|
| **CA** | Arithmétique Centrale | Effectue +, -, ×, ÷. Équivalent des neurones associatifs. |
| **CC** | Contrôle Central | Séquence les opérations. Contient les instructions définissant une tâche particulière. |
| **M** | Mémoire | Stocke données et instructions. Mémoire totale de l'appareil. |
| **I** | Entrée (Input) | Transfère depuis le support externe R vers M. Toujours via M, **jamais directement vers C**. |
| **O** | Sortie (Output) | Transfère depuis M vers le support R. Toujours depuis M, **jamais directement depuis C**. |

> **Important :** Aujourd'hui, pratiquement tous les ordinateurs ont cette même structure générale et fonctionnement. Ils sont donc appelés **« machines de von Neumann »**. La spécialisation se fait au moment de la **PROGRAMMATION**, pas au moment de la **CONCEPTION**.

---

## Structure de l'IAS

L'ordinateur IAS comprend les composants suivants :

- Une **mémoire principale**, qui stocke à la fois données et instructions
- Une **ALU** (*Arithmetic and Logic Unit*) capable de traiter des données binaires
- Une **unité de contrôle**, qui interprète les instructions en mémoire et en provoque l'exécution
- Des **équipements d'entrée/sortie (I/O)** opérés par l'unité de contrôle

---

## Organisation de la mémoire de l'IAS

La mémoire de l'IAS se compose de **4 096 emplacements** de stockage (mots/*words*) de **40 bits** binaires chacun. Il existe deux types de mots :

### (a) Mot numérique (40 bits)
```
┌───────┬────────────────────────────────────────┐
│ Signe │           Valeur numérique             │
│ 1 bit │              39 bits                   │
└───────┴────────────────────────────────────────┘
                  ← 40 bits au total →
```

### (b) Mot d'instruction (40 bits = 2 instructions de 20 bits)
```
┌──────────────┬────────────────┬──────────────┬────────────────┐
│ Opcode gauche│ Adresse gauche │ Opcode droit │ Adresse droite │
│    8 bits    │    12 bits     │    8 bits    │    12 bits     │
└──────────────┴────────────────┴──────────────┴────────────────┘
│←──── Instruction gauche (bits 0–19) ────→│←── Instr. droite (bits 20–39) ──→│
```

- **Opcode** : 8 bits → 256 opcodes possibles (21 utilisés)
- **Adresse** : 12 bits → 4 096 (2¹²) emplacements mémoire

---

## Les registres de l'IAS

L'unité de contrôle et l'ALU contiennent des emplacements de stockage appelés **registres** :

| Registre | Nom complet | Taille | Fonction détaillée |
|---|---|---|---|
| **MBR** | Memory Buffer Register | 40 bits | Interface entre CPU et mémoire. Contient un mot à écrire en mémoire ou reçu depuis la mémoire. |
| **MAR** | Memory Address Register | 12 bits | Spécifie l'adresse en mémoire du mot à lire ou écrire. Pointeur vers la mémoire. |
| **IR** | Instruction Register | 8 bits | Contient l'opcode (8 bits) de l'instruction en cours d'exécution. |
| **IBR** | Instruction Buffer Register | 20 bits | Maintient temporairement l'instruction de droite d'un mot mémoire (pendant que la gauche est exécutée). |
| **PC** | Program Counter | 12 bits | Contient l'adresse de la prochaine paire d'instructions à récupérer. Le **compteur de programme**. |
| **AC** | Accumulator | 40 bits | Maintient temporairement les opérandes et résultats ALU. Pour une multiplication (résultat 80 bits) : partie **haute** dans AC. |
| **MQ** | Multiplier Quotient | 40 bits | Complète l'AC. Pour une multiplication : stocke les 40 bits de **poids faible**. Utilisé aussi pour les divisions. |

---

## Le cycle d'instruction de l'IAS

L'IAS fonctionne en exécutant répétitivement un **cycle d'instruction** composé de deux sous-cycles :

```
┌──────────────────────────────────────────────────────────────────┐
│               CYCLE D'INSTRUCTION (répété en boucle)             │
├──────────────────────────────────┬───────────────────────────────┤
│   SOUS-CYCLE 1 : FETCH           │   SOUS-CYCLE 2 : EXECUTE      │
│   (Récupération)                 │   (Exécution)                 │
├──────────────────────────────────┼───────────────────────────────┤
│ Étape 1 : L'opcode de la         │ Étape 1 : Les circuits de     │
│ prochaine instruction est        │ contrôle interprètent         │
│ chargé dans IR                   │ l'opcode dans IR              │
│                                  │                               │
│ Étape 2 : La partie adresse est  │ Étape 2 : Envoi des signaux   │
│ chargée dans MAR                 │ de contrôle appropriés        │
│                                  │                               │
│ Source : IBR (si instr. droite   │ Résultat : déplacement de     │
│ disponible) ou mémoire           │ données OU opération ALU      │
│ (MAR←PC, MBR←M[MAR])            │                               │
└──────────────────────────────────┴───────────────────────────────┘
```

> **Pourquoi l'indirection via MBR/MAR ?** Pour simplifier les circuits électroniques : un seul registre pour l'adresse (MAR), un seul pour les données (MBR).

---

## Le jeu d'instructions de l'IAS (21 instructions)

L'IAS avait au total **21 instructions**, regroupées en 5 catégories :

| Catégorie | Nombre | Description | Exemples |
|---|---|---|---|
| **Transfert de données** | 7 | Déplace des données entre mémoire et registres ALU, ou entre registres | `LOAD M(X)`, `STOR M(X)`, `LOAD MQ` |
| **Branchement inconditionnel** | 2 | Change la séquence d'exécution vers n'importe quelle instruction | `JUMP M(X, 0:19)`, `JUMP M(X, 20:39)` |
| **Branchement conditionnel** | 2 | Le branchement dépend d'une condition (AC ≥ 0) | `JUMP+ M(X, 0:19)` |
| **Arithmétique** | 8 | Opérations ALU : addition, soustraction, multiplication, division, décalages | `ADD M(X)`, `SUB M(X)`, `MUL M(X)`, `DIV M(X)`, `LSH`, `RSH` |
| **Modification d'adresse** | 2 | Calcule des adresses dans l'ALU et les insère dans les instructions en mémoire | `STOR M(X, 8:19)`, `STOR M(X, 28:39)` |

---

## Exemple de fonctionnement : l'opération de multiplication

La multiplication sur l'IAS illustre la complexité des opérations même simples.

- L'instruction `MUL M(X)` nécessite **39 sous-opérations** (une par position de bit, sauf le bit de signe)
- Le résultat d'une multiplication de deux nombres 40 bits est un **nombre 80 bits** :
  - Les **40 bits de poids fort** → stockés dans l'accumulateur **AC**
  - Les **40 bits de poids faible** → stockés dans le registre **MQ**

---

> **Point clé :**
> - L'ordinateur IAS est le **prototype de TOUS les ordinateurs modernes**.
> - Le **concept de programme stocké** (données et instructions dans la même mémoire) est la base de l'architecture de von Neumann.
> - La distinction entre **fetch** et **execute** reste valable dans tous les processeurs modernes, même si le pipeline rend la frontière plus floue.
