# 1.2 — Structure et Fonction d'un Ordinateur

> Un ordinateur est un système d'une complexité extraordinaire. Les processeurs contemporains contiennent des **milliards de composants électroniques élémentaires**. Pour comprendre un tel système, il faut adopter une approche méthodique. La clé est de reconnaître la **nature hiérarchique** des systèmes complexes.

---

## L'approche hiérarchique

Un **système hiérarchique** est un ensemble de sous-systèmes interdépendants. Chaque sous-système peut à son tour contenir des sous-systèmes de niveau inférieur, jusqu'à atteindre un niveau élémentaire.

Cette hiérarchie est essentielle pour deux raisons :
- **Pour la conception** : le concepteur ne traite qu'un niveau particulier à la fois
- **Pour la description** : on peut décrire chaque niveau indépendamment des autres

### La hiérarchie d'un ordinateur (vue simplifiée)

```
┌─────────────────────────────────────────────────────┐
│           NIVEAU 1 : L'ORDINATEUR COMPLET           │
│   CPU       │   Mémoire Principale  │  I/O + Bus    │
└─────────────────────┬───────────────────────────────┘
                      │ Zoom sur le CPU
┌─────────────────────▼───────────────────────────────┐
│                  NIVEAU 2 : LE CPU                  │
│  Unité de Contrôle │    ALU    │ Registres │ Bus Int │
└─────────────────────┬───────────────────────────────┘
                      │ Zoom sur l'Unité de Contrôle
┌─────────────────────▼───────────────────────────────┐
│       NIVEAU 3 : UNITÉ DE CONTRÔLE (microprogrammée)│
│  Logique Séquencement │ Registres & Décodeurs │ Mém. │
└─────────────────────────────────────────────────────┘
```

---

## Les deux dimensions d'analyse : Structure et Fonction

| Dimension | Définition | Question posée | Exemple |
|---|---|---|---|
| **Structure** | La manière dont les composants sont interconnectés | *Comment sont-ils reliés ?* | Le CPU est connecté à la mémoire via le bus système |
| **Fonction** | Le comportement de chaque composant individuel au sein de la structure | *Qu'est-ce que ça fait ?* | La mémoire stocke les données ; le CPU les traite |

---

## Les quatre fonctions fondamentales d'un ordinateur

Malgré leur complexité apparente, tous les ordinateurs ne peuvent effectuer que **quatre opérations fondamentales** :

### 1. 🔧 Traitement de données
Exécuter des opérations arithmétiques, logiques, de comparaison sur les données. Les méthodes fondamentales de traitement sont en nombre limité malgré la diversité des formes de données.

### 2. 💾 Stockage de données
- **Court terme** : mémoire de travail (RAM) pour les données en cours de traitement.
- **Long terme** : fichiers stockés pour récupération et mise à jour ultérieures.

### 3. 🔄 Mouvement de données
- Entre composants internes → via **bus**
- Vers/depuis périphériques → **I/O**
- Sur de longues distances → **communications de données**

### 4. 🎛️ Contrôle
L'unité de contrôle gère les ressources de l'ordinateur et orchestre le fonctionnement de toutes les parties fonctionnelles **en réponse aux instructions**.

> **Important :** Il y a étonnamment peu d'adaptation de la structure de l'ordinateur à la fonction à accomplir. La raison profonde est la nature **polyvalente (general-purpose)** des ordinateurs : toute la spécialisation fonctionnelle se produit au moment de la **programmation**, et non au moment de la **conception**.

## Structure d'un ordinateur monoprocesseur (vue traditionnelle)

Un ordinateur monoprocesseur traditionnel se compose de **4 composants structurels principaux** :

```
┌───────────────────────────────────────────────────────────┐
│                        ORDINATEUR                         │
├─────────────────┬──────────────────┬─────────┬────────────┤
│      CPU        │ MÉMOIRE PRINCIP. │   I/O   │ BUS SYSTME │
│ (Processeur     │ (Stockage des    │ (E/S)   │(Interconn.)│
│  Central)       │  données)        │         │            │
├─────────────────┼──────────────────┼─────────┼────────────┤
│ Contrôle +      │   RAM, ROM       │Clavier, │ Adresses,  │
│ Traitement      │                  │ Écran,  │ Données,   │
│                 │                  │ Disques │ Contrôle   │
└─────────────────┴──────────────────┴─────────┴────────────┘
            ← Le BUS SYSTÈME relie tous les composants →
```

## Structure interne du CPU

Le **CPU** (Central Processing Unit) est le composant le plus complexe. Ses composantes structurelles majeures sont :

```
┌───────────────────────────────────────────────────────────────┐
│                             CPU                               │
├──────────────────┬──────────────────────────┬─────────────────┤
│  UNITÉ DE        │  ALU                     │   REGISTRES     │
│  CONTRÔLE        │  (Arithmétique & Logique)│                 │
│                  │                          │                 │
│ Contrôle l'opér. │ Effectue : +, -, ×, ÷,  │ Stockage interne│
│ du CPU et de     │ AND, OR, NOT…            │ ultra-rapide au │
│ tout l'ordinat.  │                          │ sein du CPU     │
└──────────────────┴──────────────────────────┴─────────────────┘
        ← BUS INTERNE DU CPU (interconnexion) →
```

### Détail de l'Unité de Contrôle Microprogrammée

Quand l'unité de contrôle est de type **microprogrammé**, elle exécute elle-même des micro-instructions qui définissent son fonctionnement. Sa structure interne comprend :

| Composant | Rôle |
|---|---|
| **Logique de séquencement** | Détermine l'ordre des micro-opérations |
| **Registres et décodeurs** | Registres temporaires pour le contrôle, décodage des opcodes |
| **Mémoire de contrôle** | Stocke les micro-programmes (microcode) |

## Structure d'un ordinateur multicœur (multicore)

### Définitions précises

| Terme | Définition |
|---|---|
| **CPU** (Central Processing Unit) | Portion de l'ordinateur qui récupère (*fetch*) et exécute les instructions. Comprend une ALU, une unité de contrôle et des registres. |
| **Cœur** (Core) | Unité de traitement individuelle sur une puce processeur. Peut être équivalent en fonctionnalité à un CPU sur un système monoprocesseur. |
| **Processeur** (Processor) | Pièce physique de silicium contenant un ou plusieurs cœurs. S'il contient plusieurs cœurs : **processeur multicœur**. |

### Structure d'un ordinateur multicœur typique

```
┌────────────────────────── CARTE MÈRE ──────────────────────────┐
│  Puces mémoire (RAM)  │  Puce processeur (multicore)  │  I/O  │
└───────────────────────────────┬────────────────────────────────┘
                                │ Zoom
┌───────────────────────────────▼────────────────────────────────┐
│                  PUCE PROCESSEUR MULTICŒUR                     │
│  CŒUR 0  │  CŒUR 1  │  CACHE L3 PARTAGÉ  │  CŒUR 2  │ CŒUR 3 │
└────────────────────────────────┬───────────────────────────────┘
                                 │ Zoom
┌────────────────────────────────▼───────────────────────────────┐
│                     STRUCTURE D'UN CŒUR                        │
│  Logique d'instructions  │  ALU  │  Load/Store  │  Cache L1/L2 │
└────────────────────────────────────────────────────────────────┘
```

## La mémoire cache : hiérarchie des processeurs modernes

Une caractéristique importante des ordinateurs contemporains est l'utilisation de **plusieurs couches de mémoire cache** entre le processeur et la mémoire principale :

| Niveau | Capacité | Latence |
|---|---|---|
| **Registres du CPU** | Quelques dizaines | < 1 ns |
| **Cache L1** (instruction + données) | ~32 KB par cœur | ~1–4 cycles |
| **Cache L2** | ~256 KB par cœur | ~10 cycles |
| **Cache L3** (partagé entre cœurs) | ~8–32 MB | ~40 cycles |
| **Mémoire principale (RAM)** | ~8–64 GB | ~100–200 cycles |
| **Stockage (SSD/HDD)** | Téraoctets | milliers à millions de cycles |

> *← Plus petit, plus rapide, plus cher | Plus grand, plus lent, moins cher →*

## Exemple réel : l'IBM z13

La puce processeur de l'IBM z13 contient **3,99 milliards de transistors** et comprend :
- **8 cœurs** de traitement
- Un **cache L3** substantiel partagé entre tous les cœurs
- **Logique de contrôle L3** (gestion du trafic entre L3, cœurs et environnement externe)
- **Logique de contrôle de stockage (SC)** entre les cœurs et le cache L3
- **Contrôleur mémoire (MC)** pour l'accès à la mémoire externe
- **Bus I/O GX** pour l'interface avec les adaptateurs de canaux

### Structure interne d'un cœur IBM z13

| Sous-unité | Sigle | Fonction détaillée |
|---|---|---|
| Instruction Sequence Unit | **ISU** | Détermine l'ordre d'exécution ; gère le pipeline Out-of-Order (OOO), le renommage des registres, les dépendances |
| Instruction Fetch & Branch | **IFB** | Cache d'instructions 128 KB, logique de prédiction de branchements |
| Instruction Cache & Merge | **ICM** | Travaille avec l'IFB pour gérer le cache d'instructions |
| Instruction Decode Unit | **IDU** | Analyse (parsing) et décodage de tous les opcodes de l'architecture z |
| Load-Store Unit | **LSU** | Cache de données L1 de 96 KB ; gère le trafic entre cache L2 et unités d'exécution |
| Translation Unit | **XU** | Traduit adresses logiques en adresses physiques ; contient un **TLB** |
| Core Pervasive Unit | **PC** | Instrumentation et collecte d'erreurs |
| Fixed-Point Unit | **FXU** | Opérations arithmétiques en nombres entiers |
| Vector & Floating-Point Unit | **VFU** | Opérations en virgule flottante (binaire et décimale) et vectorielles |
| Recovery Unit | **RU** | Maintient une copie de l'état du système ; gère les actions de récupération |
| Dedicated Co-Processor | **COP** | Compression de données et chiffrement pour chaque cœur |

---

> **Point clé :**
> - La **structure hiérarchique** permet de comprendre un système immensément complexe en le décomposant niveau par niveau.
> - L'approche **top-down** est la plus efficace : partir du niveau le plus élevé (l'ordinateur complet) et descendre progressivement.
> - À chaque niveau, on s'intéresse à la **STRUCTURE** (comment les composants sont reliés) et à la **FONCTION** (ce que chaque composant fait).
